Мегафон - Телеком API : Отслеживание и обработка событий

Created by Alexey Brykov, last modified by Sergei Naumov on Nov 21, 2018

Все события

Ниже приведены события, обработку которых приложение осуществляет после выполнения команд раздела "Команды"

Имя

Описание

OnIncomingCall

Событие отправляется после получения 200 Ок

OnAnswer

Событие отправляется после получения ACKот абонента

OnNoAnswer

Событие отправляется после получения Non-200 Ок от абонента

OnDisconnect

Событие отправляется после получения BYEот абонента

OnDTMF

Событие отправляется после получения DTMF от абонента

OnSpeech

Событие отправляется после определения голоса абонента

OnSilence

Событие отправляется после определения отсутствия голоса

OnQueue

Событие отправляется после постановки вызова в очередь

OnDequeue

Событие отправляется после начала обслуживания вызова

OnQueueUpdate

Событие отправляется при изменении состояния очереди


Параметры, получаемые в сообщении События:

параметрзначение
sessionУникальный номер сессии, используемый при  работе с вызовом. Тип – UUID4
anumА – номер (номер вызывающего абонента) в формате E.164, например - “79281234567”. Данный номер соответствует MSISDN API-подписки, передаваемы при аутентификации и авторизации разработчика для использования API (см. п. “Авторизация”)
bnumB – номер (номер вызываемого абонента) в формате E.164, например “79267654321”. В настоящее время допускаются следующие типы вызовов:
  • местный вызов на мобильного абонента
  • местный вызов на стационарного абонента
  • междугородний вызов на мобильного абонента
  • междугородний вызов на стационарного абонента
imsiSubsriber Identity Module -  идентификатор сим-карты пользователя
DTMFnumberDTMF - строка, принятая от абонента (клавиша, нажатая на цифровой клавиатуре)
